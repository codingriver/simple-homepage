# 超短口令版 RULES

> 目的：每次新聊天只说一句话，就能快速指定工作模式。

---

## 一、总开关口令

### 通用严谨模式

```text
按 base 执行：先检查，后修改，再验证；未验证不要说完成。
```

这句适合大多数任务，是默认推荐口令。

---

## 二、场景口令

### 1. 代码修复口令

```text
按 base + fix 执行。
```

等价含义：
- 先定位根因
- 再改代码
- 改后做实际验证
- 没验证通过不要说修好了

---

### 2. 文档全量更新口令

```text
按 base + docs 执行。
```

等价含义：
- 先全量扫描文档
- 列清单
- 再修改
- 修改后复扫残留
- 没复扫不要说全部更新完

---

### 3. 发布 / Docker Hub / 部署口令

```text
按 base + release 执行。
```

等价含义：
- 先查网络 / 登录 / 权限 / 目标资源
- 再执行发布或删除
- 最后复查远端状态
- 没复查不要说发布成功

---

## 三、更强硬的一句话版本

### 最严格通用口令

```text
严格模式：先查证据和边界，再执行，再验证；没有验证证据，不要宣布完成。
```

这句适合高风险任务，比如：
- 修线上问题
- 全量文档替换
- Docker Hub 推送
- 删除远程 tag
- 服务器部署

---

## 四、懒人版最推荐口令

如果你只想记一句，推荐永远用这句：

```text
按 base 执行：先检查，后修改，再验证；未验证不要说完成。
```
