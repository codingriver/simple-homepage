# 新聊天如何调用 RULES

## 推荐方案

采用“两层规则”最稳：

1. `base` 作为基础规则，长期默认遵守
2. 其他场景规则按任务点名启用：
   - `fix`
   - `docs`
   - `release`

---

## 我能不能指定这次聊天哪些规则生效？

可以，推荐你在新聊天开头直接点名：

### 只启用基础 + 修复规则

```text
这次按 base + fix 执行。
```

### 只启用基础 + 文档规则

```text
这次按 base + docs 执行。
```

### 只启用基础 + 发布规则

```text
这次按 base + release 执行。
```

### 明确限制只用这些规则

```text
这次聊天只按 base + docs 执行，其他规则不要扩展。
```

---

## 推荐话术

### 代码修复

```text
这次按 base + fix 执行。
```

### 文档全量更新

```text
这次按 base + docs 执行。
```

### 发布 / Docker Hub / 部署

```text
这次按 base + release 执行。
```

### 只想强调核心纪律

```text
按 base 执行：先检查，后修改，再验证；未验证不要说完成。
```

---

## 最佳实践

- 不需要每次贴整篇规则
- 新聊天时说一句启用哪套规则即可
- 复杂任务建议同时点名：
  - `base`
  - 1 个场景规则

不建议每次把所有规则都同时激活，否则容易分散重点。
