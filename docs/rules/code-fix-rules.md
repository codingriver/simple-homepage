# code-fix-rules

> 适用于 bug 修复、链路排查、回归验证。

## 执行流程

1. 先定位根因，再修改。
2. 如果用户提到具体文件，必须先阅读该文件。
3. 修改后必须做实际验证：
   - 能跑测试就跑测试
   - 能走真实链路就走真实链路
   - 能做命令验证就做命令验证
4. 没有完成实际验证前，不得说“问题已解决”。

## 回复要求

回复时必须明确写清：

- 根因判断
- 修改内容
- 已验证内容
- 尚未验证内容
- 风险点

## 推荐表述

- `已修改，待验证`
- `当前根因判断为...`
- `我已完成以下验证...`
- `以下部分还没有实际验证...`

## 禁止表述

- `已经修好了`（除非有验证证据）
- `应该没问题了`
- `大概率解决了，所以可以认为完成`
