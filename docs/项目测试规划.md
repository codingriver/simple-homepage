### 6.1 CLI 测试要测什么

CLI 测试不是简单执行一次脚本，而是要锁住脚本最关键的可回归契约：

| 脚本 | 必测内容 | 说明 |
|---|---|---|
| `manage_users.php` | 非 CLI 访问返回 403 | 先锁住“只能命令行运行”这个边界 |
| `manage_users.php` | `list` 正常输出 | 验证输出格式与空用户/有用户场景 |
| `manage_users.php` | `info <user>` 成功与不存在用户失败 | 验证参数读取与错误输出 |
| `manage_users.php` | `add <user> <pwd>` 成功、用户名非法、密码过短 | 锁住输入校验 |
| `manage_users.php` | `passwd <user> <pwd>` 成功、用户不存在、密码非法 | 锁住修改密码分支 |
| `manage_users.php` | `del <user>` 删除成功 / 不存在用户 / 最后管理员限制 | 需要确认是否通过 CLI 也受保护 |
| `manage_users.php` | `reset` 的关键副作用 | 用户清空、安装锁删除、配置重置、日志清空、代理配置占位文件重建 |
| `cli/alidns_sync.php` | 缺 AccessKey / 缺域名时报错退出 | 验证缺配置分支与退出码 |
| `cli/alidns_sync.php` | 第三方接口失败时 stderr 与退出码 | 锁住外部依赖失败反馈 |
| `cli/alidns_sync.php` | 成功时 stdout 格式与 `last_sync_at` 落地 | 验证真实副作用 |
| `cli/ddns_sync.php` | 指定任务 ID 成功 / 失败 / 不存在 | 锁住单任务入口 |
| `cli/ddns_sync.php` | 不传 ID 时 due tasks 批量执行 | 验证遍历调度行为 |
| `cli/ddns_sync.php` | 成功/部分失败/全部失败退出码 | 这是 cron 场景最关键契约 |
| `cli/run_scheduled_task.php` | 合法任务执行 | 验证调度器入口 |
| `cli/run_scheduled_task.php` | 缺 ID / 非法 ID / 不存在 ID | 需要明确当前行为并锁成回归断言 |

CLI 测试的断言维度建议统一包含：

- 退出码是否正确
- stdout / stderr 是否正确
- 数据文件是否按预期被修改
- 失败时是否只影响预期文件，不产生额外副作用
- 对外部依赖失败时，错误是否可诊断
