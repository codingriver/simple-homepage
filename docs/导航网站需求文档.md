# 导航网站需求文档 v2.4

定位：纯粹的个人私有网站导航，带账户保护，支持反向代理站点接入。  
技术栈：PHP 7.0+ (兼容至 8.x) + Nginx + JSON 文件存储，无需数据库。

---

## 一、首次安装

- [x] 检测是否首次部署：`.installed` 锁文件 + `users.json` 双重判断
- [x] 首次访问任意页面自动跳转安装向导 `/setup.php`
- [x] 安装向导表单：管理员用户名、密码、确认密码、站点名称、导航站域名（选填）
- [x] 表单校验：用户名格式（字母数字下划线横杠，2-32位）、密码长度（≥8位）、两次密码一致
- [x] **`AUTH_SECRET_KEY` 由安装向导自动生成随机密钥（64位hex），写入 `shared/auth.php`，无需手动修改**
- [x] 完成后写入 `data/users.json` + `data/config.json` + `data/.installed`
- [x] `config.json` 初始写入时包含默认值：`cookie_secure=off`、`cookie_domain=''`
- [x] ~~完成后自动登录，跳转导航首页~~ → 改为跳转登录页（密钥更换后旧Token失效，自动登录会失败，故改为让用户手动登录）
- [x] 安装完成页面内直接显示 Nginx 配置建议（含反代模板），无需再去后台找
- [x] `setup.php` 安装完成后对该页面的后续访问返回 404，防止攻击者探测安装状态

## 二、认证系统

- [x] 登录页：用户名 + 密码 + 记住我（60天）
- [x] 登录成功：写 Cookie（HttpOnly + SameSite=Lax），跳首页或 redirect 参数指定页面
- [x] **Cookie Secure 属性根据配置决定（见七章系统设置），不硬编码**
- [x] **redirect 参数安全校验：允许相对路径、同域名、私有 IP（内网部署兼容）、`.local/.lan/.internal` 等内网域名**
- [x] 登录失败：统一提示"用户名或密码错误"，不暴露具体原因
- [x] IP 登录失败锁定：连续失败 5 次，锁定 15 分钟
- [x] 登录时顺带清理 `ip_locks.json` 中已过期的锁定记录
- [x] Token：HMAC-SHA256 签名，payload 含 username/role/iat/exp/jti
- [x] 有效期：默认 8 小时；记住我延长至 60 天
- [x] **Cookie 域：由后台 `cookie_domain` 配置项决定，默认为空（绑定当前访问 host），填写 `.yourdomain.com` 可实现跨子域 SSO**
- [x] **IP 地址访问时（`HTTP_HOST` 为 IP 格式），Cookie `secure` 和 `domain` 自动降级（secure=false, domain=空），保证内网 IP 访问始终可登录，不受后台配置影响**
- [x] 退出登录：清除 Cookie，跳转登录页
- [x] `users.json` 为空但 `.installed` 存在时，登录页底部显示自救命令提示
- [x] Nginx `auth_request` 验证接口（`/auth/verify.php`，仅允许 Nginx internal 调用）
- [x] 子站中间件（`subsite-middleware/auth_check.php`，供其他 PHP 子站 require 接入）
- [x] 登录日志写入 `data/logs/auth.log`，记录 SUCCESS / FAIL / IP_LOCKED，含时间戳和 IP
- [x] **登录/登出/安装等事件自动触发 Webhook 通知**（`auth_write_log` 调用 `webhook_send`，使用 `function_exists` 兼容未加载场景）

## 四、站点跳转逻辑

- [x] Internal 类型：在原始 URL 后附加 `?_nav_token=TOKEN` 参数
- [x] Proxy 路径模式：跳转到 `/p/{slug}/`（由 Nginx 反代到内网服务）
- [x] Proxy 子域模式：跳转到 `https://{proxy_domain}/`
- [x] External 类型：直接跳转原始 URL
- [x] 所有外部链接均使用 `target="_blank" rel="noopener noreferrer"`

## 七、后台 — 系统设置

- [x] 站点名称
- [x] 导航站域名（`nav_domain`，用于生成 Nginx 配置模板）
- [x] 自定义背景：背景色（色值输入）或背景图上传（存 `data/bg/`）

  - `auto`：自动检测协议，HTTPS 时开启，HTTP 时关闭，兼容 VPS-A 网关反代（`X-Forwarded-Proto`）

- [x] **Cookie Domain（跨子域 SSO）：**
  - 留空（默认）：Cookie 绑定当前访问 host

- [x] 导出配置（浏览器下载，含 sites + config 完整格式）

- [x] 手动触发备份
- [x] 生成并下载 Nginx 反代配置文件
- [x] 查看登录日志（`auth.log` 最近 100 条，支持翻页）
- [x] **站点卡片外观设置**：布局、方向、卡片尺寸、高度、是否显示描述
- [x] **display_errors 调试开关**：后台一键开启/关闭 PHP 错误显示，自动重启 PHP-FPM
- [x] **Nginx/PHP-FPM 日志查看**：在线查看 access/error/php-fpm 日志，支持选行数、清空
- [x] **Webhook 通知配置**：支持 Telegram Bot、飞书、钉钉、自定义 POST JSON 四种类型
  - 可订阅事件：SUCCESS / FAIL / IP_LOCKED / LOGOUT / SETUP
  - 支持测试发送
  - 登录/登出/安装事件在 `auth_write_log` 中自动触发
- [x] **反代参数模板选择**：精简模式（14条参数，超时60s）/ 完整模式（60+条参数，超时86400s）

## 附录A：Cookie 配置说明

### Cookie Secure 三档模式

| 模式 | 行为 | 适用场景 |
|------|------|----------|
| `off`（默认）| HTTP/HTTPS 均可发送 | 内网调试、IP 访问、首次部署 |
| `auto` | 自动检测（含 X-Forwarded-Proto）| VPS 反代、混合环境 |
| `on` | Cookie 仅 HTTPS 发送 | 生产环境全程 HTTPS |

### Cookie Domain 与跨子域 SSO

| Cookie Domain 设置 | 效果 |
|-------------------|------|
| 留空（默认）| Cookie 绑定当前 host，无 SSO |
| `.yourdomain.com` | Cookie 共享所有子域，实现 SSO |

> IP 访问时代码自动降级：secure=false，domain=空，保证内网应急访问始终可用。

---

## 附录B：部署方式对比

| 部署方式 | Cookie Secure | Cookie Domain | 跨子域 SSO |
|---------|--------------|--------------|------------|
| IP 直接访问 | off | 空 | ❌ |
| 单域名 HTTP | off | 空或域名 | ❌ |
| 单域名 HTTPS | auto/on | 空或域名 | ❌ |
| 多子域 HTTPS+SSO | auto/on | `.yourdomain.com` | ✅ |
| VPS-A 网关+VPS-B | auto | 按需 | 按需 |
