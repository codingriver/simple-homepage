# RiverOps 前置网关部署指南

本文档适用于在 RiverOps 容器前增加宿主机 Nginx、Caddy 或其他反向代理，以提供域名和 HTTPS。

前置网关只负责 TLS 终止和流量转发，不参与 RiverOps 登录鉴权，也不修改响应 Cookie。

## 1. 部署结构

```text
Internet
   │ HTTPS
   ▼
Nginx / Caddy
   │ HTTP
   ▼
RiverOps :58080
```

## 2. Nginx 示例

```nginx
server {
    listen 80;
    server_name riverops.example.com;
    return 301 https://$host$request_uri;
}

server {
    listen 443 ssl http2;
    server_name riverops.example.com;

    ssl_certificate     /etc/letsencrypt/live/riverops.example.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/riverops.example.com/privkey.pem;

    location / {
        proxy_pass http://127.0.0.1:58080;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_set_header X-Forwarded-Host $host;
    }
}
```

RiverOps 依赖 `X-Forwarded-Proto` 判断 HTTPS Cookie，并通过转发头识别真实客户端 IP。只允许可信前置网关写入这些请求头。

## 3. Caddy 示例

```caddyfile
riverops.example.com {
    reverse_proxy 127.0.0.1:58080
}
```

## 4. 跨主机部署

如果网关与 RiverOps 不在同一台服务器：

```nginx
proxy_pass http://10.0.0.20:58080;
```

仅允许网关 IP 访问 RiverOps 的 58080 端口。公网环境建议使用 WireGuard、Tailscale 或其他受控私网连接两台服务器。

## 5. 连通性检查

```bash
curl -I http://127.0.0.1:58080/login.php
curl -I https://riverops.example.com/login.php
docker logs riverops
docker exec riverops tail -f /var/log/nginx/riverops.error.log
```

确认：

- `/` 跳转至 `/admin/index.php`
- 未登录后台请求跳转登录页
- 登录后写入 `riverops_session`
- HTTPS 部署时 Cookie 带 `Secure`
- 日志中的客户端 IP 来自可信代理转发链

## 6. 安全边界

- 不要让客户端直接伪造可信转发头。
- 不要在网关层绕过 RiverOps 的 `/admin/` 鉴权。
- 不要缓存登录、鉴权或后台 PHP 响应。
- 上传大小和超时限制应与容器内 Nginx/PHP 设置保持一致。
