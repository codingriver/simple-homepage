# 文件系统模块需求与设计文档

版本：v1.1  
状态：**已开发完成**  
适用范围：后台文件管理、本机文件管理、远程主机文件管理、后续与 SSH / Host-Agent 联动扩展

---

## 1. 目标

文件系统模块的目标不是简单做一个"能看目录"的页面，而是做成这个后台里的统一文件工作台，覆盖：

- 宿主机本机文件管理
- 远程主机文件管理
- 浏览器内基础编辑
- 上传、下载、压缩、解压
- 权限与属主属组管理
- 风险操作审计
- 后续与 Web 终端、计划任务、SSH 密钥、备份模块联动

最终体验目标参考 1Panel / 宝塔的文件管理，但当前阶段优先做稳定、可控、可审计、适合容器环境的版本。

---

## 2. 产品定位

文件系统模块定位为后台核心基础能力，服务以下场景：

- 在后台直接编辑配置文件，如 `nginx.conf`、`docker-compose.yml`、`sshd_config`
- 管理计划任务脚本、日志、输出文件
- 浏览和维护站点挂载目录、备份目录、证书目录
- 通过远程 SSH 管理其它 VPS / 服务器文件
- 为后续"上传部署包、解压发布、模板下发、日志排障"提供基础设施

这个模块应优先保证：

- 可控：默认只开放允许的路径与主机
- 稳定：不能因为大文件、二进制文件、失败请求把页面拖死
- 可审计：关键操作要记录
- 一致：本机与远程主机操作体验尽量统一

## 4. 使用对象与权限模型

### 4.1 用户角色

- `admin`
  - 完整文件系统权限
- `host_admin`
  - 主机与文件管理权限
- `host_viewer`
  - 只读文件浏览权限

### 4.2 建议细粒度权限

建议新增或明确以下权限点：

- `fs.view`
- `fs.read`
- `fs.write`
- `fs.delete`
- `fs.upload`
- `fs.download`
- `fs.archive`
- `fs.permission`
- `fs.remote`
- `fs.audit`

### 4.3 权限控制原则

- 没有 `fs.write` 时：
  - 不显示新建、保存、上传、重命名、移动、复制、删除按钮
- 没有 `fs.permission` 时：
  - 不显示 `chmod/chown/chgrp`
- 没有 `fs.archive` 时：
  - 不显示压缩/解压
- 没有 `fs.remote` 时：
  - 只能看本机，不显示远程主机切换

## 5. 路径与安全边界

这是文件系统模块最关键的设计点。

### 5.1 本机路径策略

本机不能默认无限制暴露整个容器文件系统，必须采用"白名单根目录"策略。

建议默认白名单：

- `/var/www/nav/data`
- `/var/www/nav/public`
- `/var/www/nav/admin`
- `/var/www/nav/shared`
- `/etc/nginx`
- `/etc/ssh`
- `/var/log`

后续在系统设置中允许管理员追加白名单路径。

### 5.2 容器与宿主机关系

当前项目运行在容器内，因此要明确：

- 默认文件管理看到的是容器视角文件系统
- 若某路径是宿主机挂载进容器的卷，则用户实际操作的是宿主机对应目录
- 若用户需要完整宿主机文件管理，应通过 `host-agent` 扩展宿主机文件能力

### 5.3 路径校验要求

- 所有本地路径必须：
  - 归一化
  - 禁止 `..` 逃逸
  - 禁止软链绕过白名单根
- 所有远程路径必须：
  - 做 shell 转义
  - 不允许原样拼接用户输入到命令

## 7. 核心交互设计

### 7.1 文件打开逻辑

- 小文本文件：直接读取并展示到编辑器
- 二进制文件：不直接编辑，只提供下载、属性、十六进制占位提示
- 大文件：超过阈值后不自动全文加载，提示用户下载或仅查看前 N KB

建议阈值：

- 文本在线编辑阈值：2 MB
- 预览阈值：512 KB

### 7.2 删除逻辑

- 删除文件直接确认
- 删除目录需二次确认，并明确是否递归
- 批量删除需显示条目数

### 7.3 上传逻辑

- 支持单文件上传
- 支持多文件上传
- 上传冲突时支持：覆盖 / 跳过 / 自动重命名

### 7.4 复制 / 移动

采用前端暂存剪贴板状态：

- `copy`: 记录源路径列表
- `cut`: 记录源路径列表与操作类型
- `paste`: 调用后端执行

后端必须再次校验：目标路径是否允许、是否会覆盖、是否跨设备

### 7.5 压缩 / 解压

- 压缩时支持选择输出文件名
- 解压时支持选择目标目录
- 检测压缩包格式并提示不支持类型

## 11. 后端实现

### 11.1 本机执行

优先使用 PHP 原生函数：`scandir`、`is_file`、`is_dir`、`filesize`、`filemtime`、`file_get_contents`、`file_put_contents`、`mkdir`、`rename`、`unlink`、`rmdir`、`chmod`、`chown`、`chgrp`。

### 11.2 远程执行

继续基于 SSH 命令执行：`find`、`stat`、`tar`、`unzip`、`chmod`、`chown`、`chgrp`、`mv`、`cp`、`rm`。

远程操作必须统一封装命令构造，禁止页面层自己拼接 shell。

### 11.3 上传下载

本机：直接 PHP 上传流写文件。

远程：小文件可走 base64 传输，大文件后续建议改为分片上传。

## 12. 风险点

### 12.1 高风险操作

删除目录、批量删除、修改权限、修改属主、覆盖已有文件、解压未知压缩包 —— 必须二次确认、记录审计、失败时返回明确错误。

### 12.2 性能风险

大目录遍历卡顿、大文件读取卡顿、远程递归搜索耗时高、远程传输 base64 内存占用高 —— 对应策略：大目录分页、大文件阈值限制、搜索限深度、上传大小限制。

### 12.3 安全风险

路径穿越、软链逃逸、远程命令注入、覆盖关键系统文件 —— 对应策略：统一路径规范化、白名单根目录限制、shell 参数全部转义、关键路径增加风险提示。
