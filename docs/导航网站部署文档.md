# 导航网站部署文档（纯 Nginx / Ubuntu / Debian）

> 适用：Ubuntu 20.04+ / Debian 11+，纯 Nginx 环境。宝塔面板用户请参考《宝塔部署文档》。

---

## 一、环境要求

### PHP 版本

| 版本 | 支持状态 | 说明 |
|------|----------|------|
| PHP 8.2 | ✅ 推荐 | 最新稳定版，长期支持 |
| PHP 8.1 | ✅ 支持 | 稳定运行 |
| PHP 8.0 | ✅ 支持 | 稳定运行 |
| PHP 7.4 | ✅ 支持 | 稳定运行 |
| PHP 7.3 | ✅ 最低支持 | `setcookie()` 数组参数（含 `samesite`）最低要求，**7.2 以下无法运行** |
| PHP 7.2 及以下 | ❌ 不支持 | `setcookie()` 数组参数语法需要 PHP 7.3+ |
| PHP 5.x | ❌ 不支持 | — |

> ✅ 原先限制最低版本为 PHP 8.0 的 `match()` 表达式已替换为数组查找写法（PHP 7.0+ 兼容）。当前最低版本由 `setcookie()` 数组参数决定为 **PHP 7.3**。
>
> ⚠️ PHP 7.3 以下（7.0/7.1/7.2）不支持 `setcookie()` 数组参数，且均已 EOL（官方停止安全更新），不建议使用。

## 二、快速部署步骤

### 第 1 步：安装系统依赖

```bash
apt update && apt upgrade -y

# 安装 Nginx + PHP 8.2（最小化安装）
apt install -y nginx php8.2-fpm

# 可选：安装 fileinfo 扩展（背景图上传增强 MIME 检测，推荐）
# apt install -y php8.2-fileinfo
```

### 第 2 步：部署项目文件

将项目压缩包上传到服务器后解压：

```bash
# 创建项目目录
mkdir -p /var/www/nav

# 解压（假设压缩包在 /tmp/nav.zip）
unzip /tmp/nav.zip -d /var/www/nav/

# 设置权限
chown -R www-data:www-data /var/www/nav
chmod -R 755 /var/www/nav

# 创建空的反代配置文件（防止 Nginx include 报错）
touch /etc/nginx/conf.d/nav-proxy.conf
```

> ✅ **无需修改任何 PHP 文件**，以下内容均由**安装向导自动完成**：
> - `AUTH_SECRET_KEY` 随机生成并写入
> - `data/backups`、`data/logs`、`data/favicon_cache`、`data/bg` 子目录创建
> - `data/` 目录权限设为 `750`，子目录权限设为 `755`
> - 初始配置文件 `config.json`、`users.json`、`sites.json` 写入
>
> ⚠️ **仍需手动执行的步骤**：`chown -R www-data:www-data /var/www/nav`（需 root 权限，PHP 无法自行变更属主）和 `touch /etc/nginx/conf.d/nav-proxy.conf`（Nginx 配置文件操作需 root）。

### 第 3 步：配置 Nginx

```bash
# 复制配置模板
cp /var/www/nav/nginx-conf/nav.conf /etc/nginx/sites-available/nav.conf

# 编辑配置，将 nav.yourdomain.com 改为你的实际域名（或先用 _ 通配）
nano /etc/nginx/sites-available/nav.conf

# 启用站点
ln -sf /etc/nginx/sites-available/nav.conf /etc/nginx/sites-enabled/nav.conf

# 验证并重载
nginx -t && systemctl reload nginx
```

**IP 直接访问时**，Nginx server_name 可填 `_`（通配），无需绑定域名即可访问。

### 第 4 步：确认 PHP-FPM 运行

```bash
systemctl enable php8.2-fpm nginx
systemctl start php8.2-fpm
```

### 第 5 步：运行安装向导（浏览器）

打开浏览器访问：

```
http://<服务器IP>           ← IP 直接访问
http://nav.yourdomain.com  ← 域名访问（需 DNS 解析）
```

系统自动跳转到安装向导，按提示填写：
1. 管理员用户名（字母数字，2-32 位）
2. 密码（至少 8 位）
3. 站点名称
4. 导航站域名（选填，用于生成 Nginx 配置模板，IP 访问可留空）
5. 点击「开始使用」

安装完成后自动跳转到登录页，登录即可使用。

## 三、IP 访问 vs 域名访问流程

### IP 直接访问流程（第一次部署推荐）

```
浏览器 http://192.168.1.100
    ↓
Nginx → PHP → index.php
    ↓
检测 data/.installed 不存在
    ↓ 自动跳转
安装向导 setup.php
    ↓ 填写信息，提交
生成 AUTH_SECRET_KEY（自动）
写入 users.json / config.json / .installed
    ↓ 跳转
login.php → 填写账密 → 登录成功
    ↓
Cookie 写入（domain=空绑定当前IP，secure=off）
    ↓
index.php 显示导航首页 ✅
```

**特点：**
- `cookie_domain` 留空 → Cookie 自动绑定当前 IP，无跨域问题
- `cookie_secure=off` → HTTP 下 Cookie 正常发送
- 无需证书、无需域名、立即可用

### 域名 + HTTPS 访问流程

```
浏览器 https://nav.yourdomain.com
    ↓
Nginx（443，含 SSL 证书）→ PHP
    ↓
同上安装/登录流程
    ↓
Cookie 写入：
    domain = .yourdomain.com（后台配置后）
    secure = on 或 auto（后台配置后）
    ↓
跨子域 SSO 生效（所有 *.yourdomain.com 共享登录态）✅
```

**从 IP 访问升级到域名访问的步骤：**
1. 申请 SSL 证书（见附录）
2. 更新 Nginx 配置加上 443 监听
3. 登录后台 → 系统设置：
   - `Cookie Secure 模式` 改为 `auto` 或 `on`
   - `Cookie Domain` 填写 `.yourdomain.com`（前面有点）
4. 退出重新登录即生效

## 五、命令行工具

```bash
php /var/www/nav/data/manage_users.php list                    # 列出所有用户
php /var/www/nav/data/manage_users.php add admin MyPass123    # 添加管理员
php /var/www/nav/data/manage_users.php passwd admin NewPass   # 修改密码
php /var/www/nav/data/manage_users.php del olduser            # 删除用户
php /var/www/nav/data/manage_users.php reset                  # 完整重置并重新进入安装向导
```

### reset 命令重置内容

执行 `reset` 命令后，以下内容会被完整清理，随后可重新运行安装向导：

| 重置项 | 说明 |
|--------|------|
| 用户数据 | `users.json` 清空 |
| 安装锁 | `.installed` 删除 |
| cookie_secure | 重置为 `off`（HTTP + IP 均可登录）|
| cookie_domain | 清空（避免旧域名配置干扰）|
| IP 锁定记录 | `ip_locks.json` 清空 |
| 登录日志 | `data/logs/auth.log` 清空 |
| 站点与分组 | `sites.json` 清空为 `{ "groups": [] }` |
| 备份文件 | `data/backups/backup_*.json` 删除 |
| 反代配置 | `nav-proxy.conf` 等 proxy 配置清空并保留占位 |
| proxy 参数模板 | 自动检查并创建 `/etc/nginx/proxy_params_full` |
| AUTH_SECRET_KEY | 自动生成新随机密钥 |

执行后直接打开浏览器访问网站，即可看到安装向导。

## 六、Nginx 反代自动管理

后台「系统设置 → Nginx 反代管理」支持一键生成配置并 reload，需配置 sudo 白名单（**只需执行一次**）：

```bash
# 自动获取 nginx 路径并写入白名单（限定只允许 -t 和 -s reload 两个参数）
NGINX_BIN=$(which nginx || echo /usr/sbin/nginx)
{
  echo "www-data ALL=(ALL) NOPASSWD: $NGINX_BIN -t"
  echo "www-data ALL=(ALL) NOPASSWD: $NGINX_BIN -s reload"
} > /etc/sudoers.d/nav-nginx
chmod 440 /etc/sudoers.d/nav-nginx

# 验证（输出 syntax is ok 则成功）
sudo -u www-data sudo -n $NGINX_BIN -t
```

> ⚠️ **安全说明**：白名单限定了具体参数（`-t` 和 `-s reload`），而非整个 `nginx` 命令，防止 `www-data` 通过 `-c` 加载任意配置文件执行提权攻击。

## 九、安全配置清单

```
□ 安装向导已完成（data/.installed 存在）
□ AUTH_SECRET_KEY 已由安装向导自动生成（非默认值）
□ data/ 目录权限为 750，外部不可读
□ shared/ 目录权限为 755，www-data 只读（安装完成后执行：chmod 555 /var/www/nav/shared）
□ PHP-FPM 以 www-data 用户运行
□ Nginx 配置中 data/ / shared/ 已 deny all
□ setup.php 安装完成后返回 404（已验证）
□ 后台已配置 Cookie Secure 模式和 Cookie Domain（HTTPS 环境）
□ Nginx 反代 sudo 白名单已配置（如需使用反代功能）
□ sudo 白名单限制了具体参数（-t 和 -s reload），而非整个 nginx 命令
□ 无前置代理时，不信任 X-Forwarded-For 头（直接公网部署场景）
```

### 各权限说明

| 目录/文件 | 建议权限 | 说明 |
|---|---|---|
| `/var/www/nav/` | `755` | Nginx/PHP 可读 |
| `/var/www/nav/data/` | `750` | 仅 www-data 可读写，Nginx deny all |
| `/var/www/nav/data/bg/` | `755` | www-data 写入背景图 |
| `/var/www/nav/data/logs/` | `750` | 日志仅内部访问 |
| `/var/www/nav/shared/` | `555`（安装后） | 安装完成后只读，防止密钥文件被覆盖 |
| `/etc/nginx/conf.d/nav-proxy.conf` | `644`，属主 `www-data` | 后台自动写入反代配置 |

> ⚠️ `shared/auth.php` 包含 `AUTH_SECRET_KEY`，安装完成后应将 `shared/` 设为只读（`chmod -R 555 /var/www/nav/shared`），防止因其他漏洞导致密钥被篡改。

## 十、常见问题

### Q: 访问时一直跳转到 setup.php

```bash
ls -la /var/www/nav/data/.installed
cat /var/www/nav/data/users.json
# 若权限问题：
chown -R www-data:www-data /var/www/nav/data
```

### Q: 登录后立即跳回登录页（Cookie 无效）

1. 后台 → 系统设置 → Cookie Secure 模式是否为 `on`，但你在用 HTTP 访问？改为 `off` 或 `auto`
2. Cookie Domain 是否填了域名，但你在用 IP 访问？清空 Cookie Domain
3. 代码自动检测：用 IP 访问时会自动降级（secure=false，domain=空），无需手动改

### Q: 忘记管理员密码

```bash
# 重置密码
php /var/www/nav/data/manage_users.php passwd admin 新密码
# 或生成随机密码
php /var/www/nav/data/manage_users.php reset admin
# 或完整重置，重走安装向导
php /var/www/nav/data/manage_users.php setup
```

### Q: Nginx include nav-proxy.conf 报错找不到文件

```bash
touch /etc/nginx/conf.d/nav-proxy.conf
nginx -t && systemctl reload nginx
```

### Q: 502 Bad Gateway

```bash
systemctl status php8.2-fpm
systemctl start php8.2-fpm
```

### Q: 上传背景图失败

```bash
chown www-data:www-data /var/www/nav/data/bg
chmod 755 /var/www/nav/data/bg
php -i | grep upload_max_filesize
```

## 附录 A：Docker 部署（无需安装 PHP/Nginx）

> Docker 方案将 Nginx + PHP 完整打包进镜像，宿主机只需安装 Docker，无需任何 PHP/Nginx 配置。详细文档见《Docker 部署文档》。

### 宿主机前置要求

| 依赖 | 版本要求 | 安装方式 |
|---|---|---|
| Docker Engine | 20.10+ | `curl -fsSL https://get.docker.com \| sh` |
| Docker Compose | v2.0+（插件形式）或 v1.29+（独立安装）| Docker 20.10+ 已内置 Compose v2 |
| 操作系统 | Linux / macOS / Windows（WSL2）| — |
| 可用内存 | 建议 256MB+ | — |
| 磁盘空间 | 镜像约 80MB + 数据目录 | — |

> ⚠️ **不需要**在宿主机安装 PHP、Nginx、php-fpm 等任何运行环境，全部由镜像内置。

### 镜像内置环境

| 组件 | 版本 | 说明 |
|---|---|---|
| 基础系统 | Alpine Linux 3.x | 极简 Linux，镜像体积最小 |
| PHP | 8.2-fpm | 官方 `php:8.2-fpm-alpine` 镜像 |
| Nginx | Alpine 仓库最新稳定版（1.24+）| 由 `apk add nginx` 安装 |
| Supervisor | Alpine 仓库最新版 | 管理 Nginx + PHP-FPM 进程 |
| PHP 扩展 `fileinfo` | 随 PHP 编译 | 背景图 MIME 检测（增强安全）|
| PHP 扩展 `session/json/hash/pcre` | 内置核心扩展 | 无需额外安装 |
| curl | Alpine 仓库 | 健康检查探针 |
| tzdata | Alpine 仓库 | 时区数据，默认 Asia/Shanghai |

### 端口与数据配置

| 配置项 | 默认值 | 修改方式 |
|---|---|---|
| 宿主机访问端口 | `8080` | `.env` 中 `NAV_PORT=xxxx` |
| 容器内 Nginx 端口 | `80`（固定）| 无需修改 |
| 数据持久化目录 | `./nav-data` | `.env` 中 `DATA_DIR=/path/to/dir` |
| 时区 | `Asia/Shanghai` | `.env` 中 `TZ=Asia/Tokyo` 等 |

### 快速启动

```bash
# 1. 进入项目目录
cd nav-portal

# 2. 方式一：一键交互脚本（推荐首次部署）

# 2. 方式二：手动 Compose
cp .env.example .env          # 复制配置模板
nano .env                      # 按需修改端口等
docker compose up -d --build   # 构建并启动
```

### 持久化说明

```
只有 data/ 目录需要挂载到宿主机：
  宿主机 ./nav-data/  ←→  容器 /var/www/nav/data/

包含：用户数据、站点配置、日志、背景图、Favicon 缓存

```

### 运维速查

```bash

```

## 附录 B：申请 SSL 证书（启用 HTTPS）

HTTPS **不是必须的**，IP 直接访问即可正常使用。需要跨子域 SSO 或公网访问时建议启用。

**前提条件：** 服务器有公网 IP，域名已解析到该 IP，80/443 端口开放。

```bash
apt install -y certbot python3-certbot-nginx
certbot --nginx -d nav.yourdomain.com
certbot renew --dry-run
```

启用 HTTPS 后，在后台 → 系统设置中：
- `Cookie Secure 模式` 改为 `auto`（推荐）或 `on`
- `Cookie Domain` 填写 `.yourdomain.com`（需要跨子域 SSO 时）
