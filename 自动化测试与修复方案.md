# AI 辅助自动化测试与修复方案

> 基于 Nav Portal 项目实战经验总结。
> 目标：让 AI 在 PHP/Docker 项目中实现「修改→同步→测试→修复」的完整自动化闭环。

---

## 背景：本次调试的卡点统计

| 卡点类型 | 次数 | 根本原因 |
|---------|------|----------|
| 文件同步失败（空文件） | 3次 | `docker cp` 超时、stdin 重定向在 docker exec 下失效 |
| 测试环境选错（容器内 curl 死锁） | 2次 | PHP-FPM worker 数量限制导致死锁 |
| BASE URL 错误 | 1次 | 测试脚本硬编码了错误地址 |
| 数据重置路径错误 | 2次 | 容器内路径 vs 宿主机路径混淆 |
| cookie 发送失败 | 1次 | curl cookie jar 的 `#HttpOnly_` 前缀导致 cookie 不被发送 |
| 文件为空未检测 | 1次 | 同步后未验证 MD5 |

**根本原因**：没有可靠的「修改→同步→验证」自动化闭环。

## 方案一：给 AI 更好的指令（零配置成本）

### 项目信息模板

在开始任何 PHP/Docker 项目时，在对话开头提供以下信息：

```
项目信息：
- 容器名：nav-portal
- 宿主机源码路径：/volume3/storage/docker/nav-portal/nav-portal
- 容器内路径：/var/www/nav
- 对外访问地址：http://192.168.2.2:58080
- 测试账号：admin / Admin@2026!
- 数据目录（容器内）：/var/www/nav/data

规则：
1. 每次修改文件后，用 base64 同步并验证 MD5
2. 测试永远从宿主机发起 HTTP 请求，不在容器内 curl 自身
3. 重置数据用 docker exec rm，不用宿主机路径
4. 同步命令：base64 <src> | docker exec -i <container> sh -c 'base64 -d > <dst>'
5. 验证命令：对比 md5sum 两端文件
```

### 效果

只需这段话，本次 90% 的来回都可以避免。适合临时项目或不想改动基础设施的场景。

## 方案二：配置自动化测试环境（一劳永逸）

### 2.1 配置 Docker Volume（最重要）

把容器内源码和数据目录挂载到宿主机，消除「两套路径」问题。

**修改 `docker-compose.yml`**：

```yaml
services:
  nav-portal:
    volumes:
      # 源码目录：宿主机修改后容器立即生效，无需 docker cp
      - ./nav-portal:/var/www/nav
      # 数据目录单独挂载，数据持久化且路径统一
      - ./data:/var/www/nav/data
```

**效果**：修改宿主机文件后容器立即生效，彻底消除同步问题。解决本次 **50% 的卡点**。

> ⚠️ 注意：挂载后 Dockerfile 里 `COPY` 的文件会被 volume 覆盖，确保宿主机目录内容完整。

---

### 2.2 配置 Makefile（标准化操作命令）

在项目根目录创建 `Makefile`：

```makefile
CONTAINER = nav-portal
SRC       = /volume3/storage/docker/nav-portal/nav-portal
BASE      = http://192.168.2.2:58080

# ── 同步文件（带 MD5 验证）──
# 用法：make sync FILES="public/login.php admin/backups.php"
sync:
	@for f in $(FILES); do \
		echo "Syncing $$f..."; \
		base64 $(SRC)/$$f | docker exec -i $(CONTAINER) sh -c "base64 -d > /var/www/nav/$$f"; \
		H1=$$(md5sum $(SRC)/$$f | cut -d' ' -f1); \
		H2=$$(docker exec $(CONTAINER) md5sum /var/www/nav/$$f | cut -d' ' -f1); \
		[ "$$H1" = "$$H2" ] && echo "  ✅ $$f" || echo "  ❌ $$f SYNC FAILED"; \
	done

# ── 同步所有 PHP 文件 ──
sync-all:
	@find $(SRC) -name '*.php' | while read f; do \
		rel=$${f#$(SRC)/}; \
		base64 $$f | docker exec -i $(CONTAINER) sh -c "base64 -d > /var/www/nav/$$rel"; \
		echo "  synced: $$rel"; \
	done
	@echo "✅ All PHP files synced"

# ── 语法检查所有 PHP 文件 ──
lint:
	@echo "Checking PHP syntax..."
	@errors=0; \
	for f in $$(find $(SRC) -name '*.php'); do \
		result=$$(php -l $$f 2>&1); \
		if echo "$$result" | grep -q 'Parse error\|Fatal error'; then \
			echo "  ❌ $$f"; echo "  $$result"; errors=$$((errors+1)); \
		fi; \
	done; \
	[ $$errors -eq 0 ] && echo "✅ No syntax errors" || echo "❌ $$errors files with errors"

# ── 重置安装状态 ──
reset:
	docker exec $(CONTAINER) rm -f \
		/var/www/nav/data/.installed \
		/var/www/nav/data/users.json \
		/var/www/nav/data/config.json \
		/var/www/nav/data/sites.json
	@echo "✅ Data reset done"

# ── 运行完整 HTTP 测试 ──
test: reset
	@python3 /tmp/http_test.py

# ── 查看 PHP 错误日志 ──
logs:
	docker exec $(CONTAINER) tail -50 /var/log/fpm-php.www.log 2>/dev/null || \
	docker exec $(CONTAINER) tail -50 /var/log/nginx/error.log

# ── 检查容器内所有 PHP 文件是否与宿主机一致 ──
diff:
	@echo "Checking file consistency..."
	@find $(SRC) -name '*.php' | while read f; do \
		rel=$${f#$(SRC)/}; \
		H1=$$(md5sum $$f | cut -d' ' -f1); \
		H2=$$(docker exec $(CONTAINER) md5sum /var/www/nav/$$rel 2>/dev/null | cut -d' ' -f1); \
		[ "$$H1" != "$$H2" ] && echo "  ❌ OUT OF SYNC: $$rel" || true; \
	done
	@echo "✅ Diff check done"

.PHONY: sync sync-all lint reset test logs diff
```

**常用命令**：
```bash
make lint              # 检查所有 PHP 语法
make sync FILES="public/login.php admin/backups.php"  # 同步指定文件
make sync-all          # 同步所有 PHP 文件
make diff              # 检查哪些文件未同步
make reset             # 重置安装状态
make test              # 重置 + 运行完整 HTTP 测试
make logs              # 查看错误日志
```

### 2.3 配置 PHP 错误日志（关键调试工具）

**修改 `docker/php-fpm.conf`**：
```ini
[www]
catch_workers_output = yes
php_admin_value[error_log] = /var/log/fpm-php.www.log
php_admin_flag[log_errors] = on
```

**修改 `docker/php.ini`**：
```ini
display_errors = Off
log_errors = On
error_log = /var/log/php/error.log
error_reporting = E_ALL
```

**效果**：本次「PHP 输出 0 字节」问题，有了错误日志 30 秒定位，而不是来回调试 20 轮。

### 2.4 配置 Cursor Rules（让 AI 自动遵守规范）

创建 `.cursor/rules/php-docker.md`：

```markdown
## PHP Docker 项目规范

### 环境信息
- 容器名：nav-portal
- 宿主机源码：/volume3/storage/docker/nav-portal/nav-portal
- 容器内路径：/var/www/nav
- 访问地址：http://192.168.2.2:58080
- 数据目录（容器内）：/var/www/nav/data

### 文件同步规则
- 修改 PHP 文件后，必须用 base64 方式同步到容器
- 同步命令：base64 <src> | docker exec -i nav-portal sh -c 'base64 -d > <dst>'
- 同步后必须验证 MD5 一致性
- 禁止使用 docker exec cat > 方式（会产生空文件）
- docker cp 不可靠，优先用 base64 方式

### 测试规范
- 所有 HTTP 测试从宿主机发起，BASE=http://192.168.2.2:58080
- 禁止在容器内 curl 容器自身（会导致 PHP-FPM worker 死锁）
- 重置数据用 docker exec rm（不用宿主机路径删除容器内文件）
- curl cookie jar 中 #HttpOnly_ 前缀的 cookie 不会被自动发送，需用 -H 'Cookie: ...' 显式传递

### PHP 开发规范
- session_start() 必须在文件最顶部，任何输出之前
- 所有 POST/重定向处理必须在 require header.php 之前
- 文件同步后检查 PHP 语法：php -l filename.php
- CSRF 保护：所有表单加 csrf_field()，所有 POST 加 csrf_check()

### 调试顺序
1. 检查文件 MD5 是否同步成功
2. 用 php -l 检查语法
3. 检查 nginx error log
4. 检查 PHP-FPM error log（/var/log/fpm-php.www.log）
5. 用 curl -v 查看完整请求/响应头
```

## 方案三：Cursor MCP 插件扩展

通过配置 MCP Server，让 AI 拥有直接操作 Docker 容器的能力，无需通过 Shell 工具中转。

### 3.1 Docker MCP Server

在 Cursor 的 MCP 配置文件（`~/.cursor/mcp.json`）中添加：

```json
{
  "mcpServers": {
    "docker": {
      "command": "npx",
      "args": ["-y", "@docker/mcp-server"],
      "env": {
        "DOCKER_HOST": "unix:///var/run/docker.sock"
      }
    }
  }
}
```

**能力**：
- 直接读取容器日志（`docker logs`）
- 在容器内执行命令（`docker exec`）
- 管理容器生命周期（`start`/`stop`/`restart`）
- 文件操作（`docker cp`）

### 3.2 HTTP 测试 MCP Server

```json
{
  "mcpServers": {
    "http-client": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-fetch"]
    }
  }
}
```

**能力**：AI 可以直接发起 HTTP 请求测试接口，无需写 Python 测试脚本。

### 3.3 自动化测试脚本常驻化

将测试脚本固定放置并注册为 MCP resource：

```bash
# 保存测试脚本到固定位置
cp /tmp/http_test.py /volume3/storage/docker/nav-portal/http_test.py
chmod +x /volume3/storage/docker/nav-portal/http_test.py
```

在 `.cursor/rules/php-docker.md` 中添加：
```markdown
### 测试脚本
- 完整 HTTP 测试脚本：/volume3/storage/docker/nav-portal/http_test.py
- 运行命令：python3 /volume3/storage/docker/nav-portal/http_test.py
- 测试前必须先 make reset 重置数据
```
